### 整体流程：
内部项目，自研：
```
需求评审----产品经理PM---需求文档/原型/需求评审结果
			技术负责人/架构师
			前后端
			测试（可选）
技术设计---技术负责人/架构师---技术方案文档/数据库设计/接口文档
			前后端
开发---后端开发---可运行代码（基本自测通过）
			前端开发
提交代码git--开发人员--合并到develop分支
			技术负责人/架构师
自动构建--运维人员（搭建CI）---jar包/docker镜像
			技术负责人（定规则）
			开发（修构建失败问题）
测试环境部署+测试---测试人员QA测试---测试报告、bug单
			运维搭建环境
			开发修复bug
预发布环境UAT部署---运维（部署uat）--uat验收通过
			产品经理（验收）
			技术负责人（确认）
生产部署--运维（上线、回滚）--正式上线系统
			技术负责人（发布决策）
			开发（现场支持）
运维监控--运维--稳定运行、新一轮需求
			产品（收集新需求）
			开发（性能优化、故障定位）

```
外包项目：
```
客户需求（口头/文档）
简要设计方案
开发
本地打包
客户测试环境
客户点点看
客户验收
客户上线
维护期支持
```

关于自动构建：
```
CI:
构建、验证
运维人员搭建CI平台，定义构建规则pipeline、添加构建触发规则，提交立即构建（最标准的CI）
CD：
部署、发布

一般公司的cicd流程：
提交代码
↓
CI：编译 + 测试 + 打包
↓
自动部署到 test
↓
测试通过
↓
手动点击：部署到 UAT
↓
UAT 验收
↓
审批
↓
手动点击：部署到 prod


```
