一、开源与闭源

闭源是指软件仅发布编译/解释后的可执行形式，  
不主动公开源代码；  
虽然理论上可逆，但在工程、时间和法律成本上不可行，  
因此在实践中等同于“无法获得源码”。

|类型|定义|
|---|---|
|**开源**|**主动公开源代码**，允许查看、修改、再分发（受协议约束）|
|**闭源**|**仅发布可执行代码**，不公开源代码，逆向成本极高|

📌 **是否“可逆”不是分类标准**



二、web服务器、应用服务器、反向代理服务器

web服务器（web server）
为万维网提供服务的服务器
	监听80/443网络请求,等待浏览器连接
	解析 HTTP 请求
	查找并返回资源,静态文件从本机返回；动态内容，求转交给application server或运行时环境，再返回
	处理错误
	安全与性能:HTTPS 加密、压缩、缓存等
常见web server
	apache
	nginx
	lls
简单来说就是返回网页

应用服务器（Application Server）
	运行后端业务逻辑（如用户登录、数据库查询），处理动态内容
	例如：Tomcat (Java), uWSGI/Gunicorn (Python), PHP-FPM, Node.js
Web 应用服务器
	常指既能处理 HTTP 请求（Web Server 功能），又能运行应用逻辑（Application Server 功能）的软件
	例如：tomcat、

反向代理服务器
	 接收客户端 HTTP/HTTPS 请求
	根据规则将请求转发给后端服务器
	将后端响应返回给客户端
	自身通常不生成或存储内容（除非做缓存）
	例如：HAProxy、

包管理工具

1.在 Linux 早期，源码安装是主流方式：
源码--用户自己编译
优点:
	灵活：可以指定安装路径、功能模块。
	最新：可以直接用最新源码。


2.二进制包（Package / 打包安装）
debian--APT / dpkg
RedHat --YUM / DNF / RPM
--编译好打包成软件包--用户使用包管理器管理软件
优点：
	依赖管理
	升级方便
	稳定性
![[Pasted image 20251221124614.png]]
3.容器化 / 自包含二进制(docker/snap/flatpak)
把软件和运行它需要的所有依赖打包在一起，形成一个自包含的镜像或包。运行时不依赖系统已有库，类似“沙箱”
优点：
	依赖隔离：每个容器或包自带依赖，不影响系统其他软件。
	版本共存：可以同时运行多个不同版本的同一软件。
	升级回滚：更新一个镜像不影响旧版本，回滚方便。
![[Pasted image 20251221124630.png]]
4.语言生态自带的包管理（Java/JAR、Python/pip 等）
高级语言提供自己的打包和管理方式，不依赖 Linux 系统包管理器
