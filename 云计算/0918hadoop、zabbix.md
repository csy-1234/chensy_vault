# 提问

###### ==集群和分布式集群是什么东西==？
🔹 **普通集群（Cluster）**

- **核心点**：多台物理机器 → 组成一个整体系统。
    
- **任务特点**：通常执行相同或相似的任务（比如 Web 请求处理、数据库读写）。
    
- **目标**：
    
    - 提高 **并发处理能力**（更多请求同时处理）。
        
    - 提高 **可靠性**（某台机器挂了，其他机器顶上）。
        
- **举例**：Web 服务器集群、数据库主从集群。
    

---

 **分布式集群（Distributed Cluster，大数据场景）**

- **建立在集群的基础上**：也是多台物理机组成一个整体系统。
    
- **分布式的含义**：
    
    - **数据分布式存储**：数据被拆分后存放在不同节点。
        
    - **计算分布式执行**：任务也被拆分，分发到对应的数据节点并行处理。
        
- **目标**：
    
    - 提高 **单个大数据集的处理能力**（而不仅仅是并发请求数）。
        
    - 解决 **单机存不下 / 算不动**的问题。
        
- **举例**：Hadoop（HDFS + MapReduce）、Spark、Flink。
    

---

✅ 总结一句话：

- **集群** = 多机 → 高并发、高可用。
    
- **分布式集群** = 集群 + 数据分布 → 面向大数据，解决存储 & 计算瓶颈。


|维度|集群|分布式集群|
|---|---|---|
|数据存储方式|完整副本（冗余）|拆分分片（每节点部分数据）|
|数据分布形式|复制分布|分片分布|
|读写方式|主库写、从库读（可选读写分离）|每节点处理自己分片的读写请求|
|目的|高可用、高并发|高可用、高并发 + 解决存储/计算瓶颈|


###### ==hadoop是什么？==
Hadoop 是一个 **开源的分布式大数据处理框架，用于存储和分析海量数据**
- **分布式** 👉 Hadoop 不是一台机器跑，而是 **多台机器组成集群**，把任务和数据分散处理。
    
- **大数据处理** 👉 能处理 **TB、PB 级别**的数据量。
什么是“框架”？👉 

- **框架** ≠ 单纯的库或者工具。
    
- 框架是别人已经设计好的一整套 **解决问题的模式、规则、工具集合**。
作用就是：
用一堆普通电脑组成集群，来存储和处理特别特别大的数据

==hedoop四大核心组件的作用？==

在 Hadoop 3.x 中：

- **HDFS** 👉 负责存数据
    
- **MapReduce** 👉 负责算数据
    
- **YARN** 👉 负责管资源、调度任务
    
- **Common** 👉 提供公共支撑

==企业实际应用是什么？==

**Hadoop 在企业里就是用来处理大数据的**，包括：

- **存储大数据**：把海量数据分布在集群里
    
- **计算处理大数据**：批量清洗、统计、分析
    
- **挖掘价值**：生成报表、推荐、预测等
    

一句话总结：

> **Hadoop = 存储 + 处理 + 分析大数据的企业级解决方案**

==zabbix是什么？==
Zabbix = 数据采集 + 存储 + 告警 + 可视化管理的完整监控系统
==zabbix搭建？==
- **Zabbix Server** 可以单独运行在 Linux 上
    
- **Zabbix Web 前端** 必须跑在 **LNMP（Linux + Nginx + MySQL + PHP）** 或 **LAMP（Linux + Apache + MySQL + PHP）** 环境
    
- 如果 Web 前端没 PHP 或数据库配置不对，就无法访问安装界面

==zabbix采集数据方式==？
Zabbix 数据采集方式

Zabbix 采集数据主要有三种方式：

| 方式                      | 说明                                                                | 是否需要 Agent                      |
| ----------------------- | ----------------------------------------------------------------- | ------------------------------- |
| **Zabbix Agent**        | 安装在被监控主机上，主动向 Zabbix Server 发送数据（主动模式）或被 Zabbix Server 拉取数据（被动模式） | ✅ 需要                            |
| **SNMP**                | 用于网络设备或服务器支持 SNMP 的情况                                             | ❌ 不需要 Zabbix Agent，但设备必须支持 SNMP |
| **IPMI / JMX / Script** | 特殊硬件或应用监控                                                         | ❌ 不需要 Agent 或需要额外组件             |

---

 Zabbix Agent 工作原理

- Agent 安装在被监控主机上（Linux/Windows）。
    
- 监听特定端口（默认 Linux 10050，Windows 10050）。
    
- 根据 Server/Proxy 的请求，采集 CPU、内存、磁盘、服务状态等数据。
    
- 数据发送给 Zabbix Server 或 Zabbix Proxy，再由前端显示。
    

---

 小结

- 只要 Zabbix Server 看到某个主机有 Agent，就能采集该主机数据。
    
- 如果没有安装 Agent（或 Agent 没有正确运行），Zabbix 就无法采集数据（除非通过 SNMP 等方式）。
