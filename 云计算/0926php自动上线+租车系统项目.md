# 租车系统.txt
三台机器：nginx----tomcat----mysql（浏览器做客户端）
![[Pasted image 20251016203107.png]]


本上就是一个典型的 **三层 Web 架构**：

- **Nginx**：反向代理 / 负载均衡 / 静态资源
    
- **Tomcat**：Java Web 应用服务器（业务逻辑层）
    
- ***n (可能是多台 Tomcat 或 Node / 应用服务器)**：中间层横向扩展
    
- **MySQL**：数据库层

| 角色                    | 主要用途                 | 推荐内存         | 推荐 CPU | 说明                   |
| --------------------- | -------------------- | ------------ | ------ | -------------------- |
| 🧩 **Nginx**          | 反向代理 / 静态资源服务        | 512 MB～1 GB  | 1 核    | Nginx 内存占用极低，1 GB 足够 |
| 🔧 **Tomcat（单台）**     | 运行 Java Web 应用       | 2 GB～4 GB    | 2 核    | Java 堆 + JVM 较耗内存    |
| ⚙️ **Tomcat 集群（2 台）** | 负载均衡 + 灾备            | 每台 2 GB～4 GB | 各 2 核  | 可水平扩展                |
| 🗄 **MySQL**          | 数据存储                 | 2 GB～4 GB    | 2 核    | 取决于数据量和查询复杂度         |
| 🧱 **管理 / 监控节点（可选）**  | Jenkins、Prometheus 等 | 1 GB～2 GB    | 1 核    | 可选                   |
